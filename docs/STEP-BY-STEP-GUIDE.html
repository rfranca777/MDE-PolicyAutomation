1. Visão Geral da Solução

❌ O Problema

Em ambientes com múltiplas subscriptions Azure, as VMs Windows aparecem no portal MDE como uma lista única, sem organização:

🏗️ Arquitetura em 3 Camadas

🔵 CAMADA 1 — Azure Policy (Infraestrutura)

Azure Policy DeployIfNotExists detecta VMs Windows sem a extensão MDEDeviceTagExtension e automaticamente deploya o script Set-MDEDeviceTag.ps1 que configura o registro Windows com o tag da subscription.

HKLM:\SOFTWARE\Policies\Microsoft\
  Windows Advanced Threat Protection\DeviceTagging
    Group = "mde-policy-{subscription-name}"
🟡 CAMADA 2 — Azure Automation (Operacional)

Automation Account com Managed Identity executa um Runbook PowerShell a cada hora que:

  • Descobre todas as VMs Azure + Azure Arc machines na subscription
  • Busca os devices correspondentes no Entra ID via Graph API
  • Adiciona os devices encontrados ao Security Group do Entra ID
🔴 CAMADA 3 — MDE Integration (Segurança)

MDE Device Groups são configurados para filtrar devices pela tag. Cada subscription tem um Device Group específico, permitindo políticas AV/ASR diferenciadas por ambiente (PROD, DEV, TEST).

12. Etapa 10 — Runbook PowerShell

O Que Acontece

Cria, importa e publica um Runbook PowerShell que executa a lógica de sincronização:

  1. Autentica via Managed Identity (Connect-AzAccount -Identity)

PowerShell — Exemplo de uso da MDE API
# 1. Obter token OAuth2 PS C:\> $body = @{ client_id = $appId client_secret = $secret scope = "https://api.securitycenter.microsoft.com/.default" grant_type = "client_credentials" } PS C:\> $token = (Invoke-RestMethod -Method POST ` -Uri "https://login.microsoftonline.com/$tenantId/oauth2/v2.0/token" ` -Body $body).access_token # 2. Listar machines com tags PS C:\> $headers = @{ Authorization = "Bearer $token" } PS C:\> Invoke-RestMethod ` -Uri "https://api.securitycenter.microsoft.com/api/machines" ` -Headers $headers | Select machineTags # 3. Adicionar tag a um device PS C:\> $body = @{ Value = "mde-policy-production"; Action = "Add" } | ConvertTo-Json PS C:\> Invoke-RestMethod -Method POST ` -Uri "https://api.securitycenter.microsoft.com/api/machines/{id}/tags" ` -Headers $headers -Body $body -ContentType "application/json" [OK] Tag adicionada com sucesso

18. Troubleshooting

❌ Problema: Tag não aparece no MDE Portal
  1. Verificar MDE Sense Agent: Get-Service -Name Sense → deve estar Running com StartType Automatic
  2. Verificar registry: Get-ItemProperty "HKLM:\...\DeviceTagging" → Group deve ter valor correto
  3. Aguardar sincronização: MDE sincroniza tags a cada 15-30 minutos. Primeira vez pode levar até 4 horas
  4. Verificar conectividade: VM precisa acessar *.securitycenter.windows.com
⚠️ Problema: Policy mostra "Non-Compliant"
  1. VM é Windows? Policy só se aplica a VMs Windows (osType == Windows)
  2. Managed Identity da Assignment: Precisa de role Virtual Machine Contributor
  3. Script acessível? Testar URL: Invoke-WebRequest -Uri $ScriptUri
  4. Forçar avaliação: az policy state trigger-scan --resource-group "rg-mde-*"
🔑 Problema: Runbook falha com erro de permissão
  1. Managed Identity ativada? Automation Account → Identity → System assigned → On
  2. RBAC Reader? Subscription → IAM → MI deve ter Reader
  3. Graph API permissions? Enterprise Apps → API permissions → Group.ReadWrite.All + Device.Read.All
  4. Módulo Az.Accounts? Automation Account → Modules → Status = Available
